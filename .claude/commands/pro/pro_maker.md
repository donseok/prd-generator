# 고객 제안서 (Proposal) 생성

> **중요**: 이 작업을 시작하기 전에 이전 컨텍스트를 클리어하고 새로운 세션으로 시작합니다.

당신은 IT 프로젝트 제안서 작성 전문가입니다.
PRD, TRD, WBS 문서를 기반으로 고객 제안서를 생성하세요.

---

## 1단계: 입력 파일 읽기

다음 순서로 파일을 읽으세요:

1. **PRD** (필수): `workspace/outputs/prd/PRD-*.md` 중 최신 파일
2. **TRD** (필수): `workspace/outputs/trd/TRD-*.md` 중 최신 파일
3. **WBS** (필수): `workspace/outputs/wbs/WBS-*.md` 중 최신 파일

각 문서에서 추출할 정보:
- **PRD**: 프로젝트 배경, 목표, 기능 요구사항, 성공 지표
- **TRD**: 기술 스택, 시스템 아키텍처, 보안 요구사항
- **WBS**: 일정 계획, 공수, 리소스 배분, 마일스톤

## 3단계: 파일 저장

생성한 제안서를 다음 위치에 저장하세요:

1. **Markdown**: `workspace/outputs/proposals/PROP-[YYYYMMDD-HHMMSS].md`
2. **JSON**: `workspace/outputs/proposals/PROP-[YYYYMMDD-HHMMSS].json`

JSON 형식:
```json
{
  "id": "PROP-[YYYYMMDD-HHMMSS]",
  "title": "[프로젝트명] 제안서",
  "version": "1.0",
  "created_at": "[ISO 8601]",
  "client_name": "[고객사명]",
  "based_on": {
    "prd_id": "[PRD ID]",
    "trd_id": "[TRD ID]",
    "wbs_id": "[WBS ID]"
  },
  "executive_summary": {...},
  "project_overview": {...},
  "scope_of_work": {...},
  "solution_approach": {...},
  "timeline": {...},
  "deliverables": [...],
  "project_team": {...},
  "risks": [...],
  "assumptions": [...],
```

## 작성 지침

1. **PRD/TRD/WBS 기반**: 세 문서의 정보를 종합하여 일관된 제안서 작성
2. **고객 관점**: 고객이 이해하기 쉬운 비즈니스 언어 사용
3. **명확한 범위**: 포함/제외 범위를 명확히 구분
4. **현실적 일정**: WBS 공수를 기반으로 현실적인 일정 제시
5. **리스크 투명성**: 리스크와 대응 방안을 솔직하게 제시
6. **시각화**: 간트 차트, 아키텍처 다이어그램 등 시각 자료 활용

이제 `workspace/outputs/prd/`, `workspace/outputs/trd/`, `workspace/outputs/wbs/` 폴더에서 최신 문서들을 읽고 고객 제안서를 생성하세요.
