# 고객 제안서 (Proposal) 생성

> **중요**: 이 작업을 시작하기 전에 이전 컨텍스트를 클리어하고 새로운 세션으로 시작합니다.

당신은 IT 프로젝트 제안서 작성 전문가입니다.
PRD, TRD, WBS 문서를 기반으로 **설득력 있는 스토리텔링 구조**의 고객 제안서를 생성하세요.

**인수**: $ARGUMENTS
- `--client "고객사명"`: 고객사 이름 지정 (기본값: "귀사")

---

## 스토리텔링 핵심 원칙

### 목적
**제안서를 읽는 사람들을 몰입시키는 효과적인 메시지 전달**

### 스토리 흐름 공식
```
[Hook] 공감 → [Problem] 문제 정의 → [Solution] 해결책 → [Proof] 신뢰 구축 → [Vision] 미래 비전 → [CTA] 행동 촉구
```

### 스토리텔링 3대 원칙
1. **고객 중심 서사**: "우리가 무엇을 하는가"가 아닌 "고객에게 무슨 가치를 주는가"
2. **감정적 연결**: 문제의 고통을 공감하고, 성공의 기쁨을 그리기
3. **논리적 설득**: 데이터와 근거로 신뢰 구축

---

## 1단계: 입력 파일 읽기

다음 순서로 파일을 읽으세요:

**읽기 우선순위 (JSON 우선):**
1. **PRD** (필수): `workspace/outputs/prd/PRD-*.json` 우선 → `PRD-*.md` 폴백
2. **TRD** (필수): `workspace/outputs/trd/TRD-*.json` 우선 → `TRD-*.md` 폴백
3. **WBS** (필수): `workspace/outputs/wbs/WBS-*.json` 우선 → `WBS-*.md` 폴백
- 파일명의 타임스탬프(YYYYMMDD-HHMMSS) 기준으로 최신 파일 판별

각 문서에서 추출할 정보:
- **PRD**: 프로젝트 배경, 목표, 기능 요구사항, 성공 지표
- **TRD**: 기술 스택, 시스템 아키텍처, 보안 요구사항
- **WBS**: 일정 계획, 공수, 리소스 배분, 마일스톤

## 작성 지침 (스토리텔링 체크리스트)

### 몰입 유도
- [ ] 첫 페이지에서 핵심 메시지가 전달되는가?
- [ ] 고객의 고통에 공감하는 표현이 있는가?
- [ ] "왜 지금 해야 하는가?"가 명확한가?

### 설득력 구축
- [ ] 문제 → 솔루션 → 가치의 흐름이 논리적인가?
- [ ] 정량적 데이터로 주장을 뒷받침하는가?
- [ ] 리스크를 투명하게 공유하는가?

### 행동 촉구
- [ ] 다음 단계가 명확한가?
- [ ] 연락처가 눈에 띄게 제시되었는가?
- [ ] 긍정적인 마무리로 동기 부여하는가?

### 작문 팁
1. **수동태 대신 능동태**: "시스템이 구축됩니다" → "우리가 시스템을 구축합니다"
2. **추상적 대신 구체적**: "효율성 향상" → "처리 시간 45분 → 20분 단축"
3. **기능 대신 가치**: "실시간 대시보드 제공" → "언제든 현황을 한눈에 파악"
4. **두려움과 희망**: 변하지 않으면의 위험 + 변했을 때의 미래

## 에러 처리

- **PRD 파일 없음**: "workspace/outputs/prd/ 폴더에 PRD 파일이 없습니다. /prd:prd-maker를 먼저 실행하세요." 표시 후 중단
- **TRD/WBS 파일 없음**: 가용한 문서만으로 제안서 생성. 기술/일정 섹션은 PRD 정보 기반 추정치 사용
- **불완전 데이터**: 가용한 정보로 제안서 작성 + assumptions에 추정치 사용 사실 명시

---

## 파이프라인

```
[입력 파일] → PRD → TRD → WBS → Proposal → PPT
                                 ^^^^^^^^
선행: PRD 필수, TRD/WBS 권장
후속: /ppt:ppt-maker (PPT 프레젠테이션 생성)
대안: python -m app.scripts.pro_maker --client "고객사명"
```

이제 `workspace/outputs/prd/`, `workspace/outputs/trd/`, `workspace/outputs/wbs/` 폴더에서 최신 문서들을 읽고 **스토리텔링 기반 고객 제안서**를 생성하세요.
