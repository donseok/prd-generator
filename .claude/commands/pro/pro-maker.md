# 고객 제안서 (Proposal) 생성

> **중요**: 이 작업을 시작하기 전에 이전 컨텍스트를 클리어하고 새로운 세션으로 시작합니다.

당신은 IT 프로젝트 제안서 작성 전문가입니다.
PRD, TRD, WBS 문서를 기반으로 **설득력 있는 스토리텔링 구조**의 고객 제안서를 생성하세요.

---

## 스토리텔링 핵심 원칙

### 목적
**제안서를 읽는 사람들을 몰입시키는 효과적인 메시지 전달**

### 스토리 흐름 공식
```
[Hook] 공감 → [Problem] 문제 정의 → [Solution] 해결책 → [Proof] 신뢰 구축 → [Vision] 미래 비전 → [CTA] 행동 촉구
```

### 스토리텔링 3대 원칙
1. **고객 중심 서사**: "우리가 무엇을 하는가"가 아닌 "고객에게 무슨 가치를 주는가"
2. **감정적 연결**: 문제의 고통을 공감하고, 성공의 기쁨을 그리기
3. **논리적 설득**: 데이터와 근거로 신뢰 구축

---

## 1단계: 입력 파일 읽기

다음 순서로 파일을 읽으세요:

1. **PRD** (필수): `workspace/outputs/prd/PRD-*.md` 중 최신 파일
2. **TRD** (필수): `workspace/outputs/trd/TRD-*.md` 중 최신 파일
3. **WBS** (필수): `workspace/outputs/wbs/WBS-*.md` 중 최신 파일

각 문서에서 추출할 정보:
- **PRD**: 프로젝트 배경, 목표, 기능 요구사항, 성공 지표
- **TRD**: 기술 스택, 시스템 아키텍처, 보안 요구사항
- **WBS**: 일정 계획, 공수, 리소스 배분, 마일스톤

## 스토리텔링 작성 팁
