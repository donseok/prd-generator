## BUG-001: 장바구니 수량 초기화 현상

### 기본 정보
- **심각도**: Critical
- **발견일**: 2026-01-10
- **보고자**: QA팀 이테스트
- **상태**: Open

### 재현 환경
- **디바이스**: iPhone 14 Pro
- **OS**: iOS 17.1
- **앱 버전**: 3.2.1

### 재현 단계
1. 상품 상세 페이지에서 수량 5개 선택
2. 장바구니에 담기 클릭
3. 장바구니 페이지로 이동
4. 앱을 백그라운드로 전환 (홈 버튼)
5. 5분 후 앱 재진입

### 예상 결과
- 장바구니에 수량 5개 유지

### 실제 결과
- 수량이 1개로 초기화됨

### 추가 정보
- 안드로이드에서는 재현 안 됨
- iOS 16에서도 발생하지 않음

### 요구사항 도출
- **REQ**: 장바구니 상태는 앱 재시작 후에도 유지되어야 함
- **REQ**: 백그라운드 전환 시 로컬 저장소에 자동 저장

## BUG-002: 검색 결과 무한 스크롤 중단

### 기본 정보
- **심각도**: Major
- **발견일**: 2026-01-12
- **보고자**: CS팀 (고객 제보)
- **상태**: In Progress

### 재현 환경
- **디바이스**: 다양한 안드로이드 기기
- **OS**: Android 12+
- **앱 버전**: 3.2.0 이상

### 재현 단계
1. 검색창에 "운동화" 입력
2. 검색 결과 스크롤 (약 100개 상품 이후)
3. 계속 스크롤 시도

### 예상 결과
- 추가 상품이 계속 로딩됨

### 실제 결과
- "더 이상 상품이 없습니다" 표시
- 실제로는 500개 이상 상품 존재

### 원인 분석 (개발팀 코멘트)
> 페이지네이션 offset 계산 오류로 보임. API 응답에서 hasNext가 잘못 반환되고 있음.

### 요구사항 도출
- **REQ**: 검색 결과 페이지네이션 정확성 보장
- **REQ**: 전체 결과 수 표시하여 사용자 인지 가능하게

## BUG-003: 결제 완료 후 주문 내역 미표시

### 기본 정보
- **심각도**: Critical
- **발견일**: 2026-01-14
- **보고자**: 결제팀 모니터링
- **상태**: Open

### 재현 환경
- **결제 수단**: 카카오페이
- **앱 버전**: 모든 버전

### 재현 단계
1. 상품 선택 후 결제 진행
2. 카카오페이 결제 완료
3. 앱으로 돌아옴
4. 주문 내역 확인

### 예상 결과
- 방금 완료한 주문이 목록에 표시됨

### 실제 결과
- 주문 내역에 표시되지 않음
- 새로고침 또는 재로그인 후 표시됨

### 영향도
- 하루 약 200건 발생 (카카오페이 결제의 5%)
- CS 문의 급증 원인

### 요구사항 도출
- **REQ**: 외부 결제 콜백 후 주문 목록 자동 갱신
- **REQ**: 결제 완료 페이지에서 주문 상세 바로 접근 가능
- **NFR**: 결제 완료 후 3초 이내 주문 목록 반영
