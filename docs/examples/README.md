# PRD Generator 테스트 예제 파일

이 폴더에는 PRD Generator를 테스트하기 위한 다양한 형식의 예제 파일들이 포함되어 있습니다.

## 📁 파일 목록

### 1. 회의록 (Meeting Notes)
**파일**: `meeting_notes_example.txt`

회의에서 논의된 요구사항을 담은 텍스트 형식의 예제입니다.
- 참석자 정보
- 안건별 논의 내용
- 기능/비기능 요구사항
- 액션 아이템

### 2. 사용자 인터뷰 (User Interview)
**파일**: `user_interview_example.md`

사용자 인터뷰를 정리한 마크다운 형식의 예제입니다.
- 인터뷰 대상 정보
- Pain Points 및 인용문
- 요구사항 도출 내용

### 3. 기능 요청서 (Feature Request)
**파일**: `feature_request_example.json`

구조화된 기능 요청 목록을 담은 JSON 형식의 예제입니다.
- 요청 ID 및 제목
- 비즈니스 가치
- 수락 기준 (Acceptance Criteria)
- 제약 조건

### 4. 이메일 스레드 (Email Thread)
**파일**: `email_thread_example.txt`

이해관계자 간 이메일 대화에서 추출할 수 있는 요구사항 예제입니다.
- 이슈 공유
- 기술 검토
- 의사결정 과정

### 5. 유저 스토리 (User Story)
**파일**: `user_story_example.md`

애자일 형식의 유저 스토리 예제입니다.
- Epic 및 Story 구조
- As a / I want / So that 형식
- Acceptance Criteria
- Story Points

### 6. 슬랙 대화 (Slack Conversation)
**파일**: `slack_conversation_example.txt`

팀 채팅에서 수집된 비정형 요구사항 예제입니다.
- 실시간 논의
- 긴급 이슈
- 브레인스토밍 내용

### 7. 기술 명세서 (Technical Spec)
**파일**: `technical_spec_example.md`

개발팀의 기술 명세서 형식의 예제입니다.
- 기능/비기능 요구사항 표
- 시스템 아키텍처
- API 명세
- 데이터 모델

### 8. 경쟁사 분석 (Competitor Analysis)
**파일**: `competitor_analysis_example.md`

경쟁사 분석에서 도출된 요구사항 예제입니다.
- 기능 비교표
- 우선 도입 필요 기능
- 차별화 전략

### 9. 버그 리포트 (Bug Report)
**파일**: `bug_report_example.md`

버그 리포트에서 도출된 요구사항 예제입니다.
- 재현 단계
- 예상/실제 결과
- 요구사항 도출

---

## 🚀 사용 방법

1. 위 파일 중 테스트하고 싶은 형식의 파일을 선택합니다.
2. PRD Generator API에 파일을 업로드하거나 텍스트를 입력합니다.
3. 생성된 PRD 문서를 확인합니다.

## 📝 참고사항

- 모든 예제는 가상의 전자상거래 플랫폼을 기반으로 작성되었습니다.
- 실제 회사나 제품과는 무관합니다.
- 한국어와 영어가 혼용된 실제 업무 환경을 반영했습니다.
