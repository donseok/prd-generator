## 1. 경영진 요약 (Executive Summary)

> **스토리 의도**: 바쁜 의사결정권자가 1분 안에 핵심을 파악하고 흥미를 갖도록

### 한눈에 보는 프로젝트

| 항목 | 내용 |
|------|------|
| **해결할 문제** | 15년 된 DOS 시스템으로 인한 수기 입력 오류와 45분 대기 시간 |
| **우리의 솔루션** | 웹 대시보드 + 모바일 앱으로 계량 프로세스 100% 디지털화 |
| **예상 기간** | 7개월 (28주) |
| **투입 공수** | 16 Man-Months |
| **핵심 기대효과** | 대기 시간 55% 단축 (45분 → 20분), 오기입 제로화 |

> **왜 지금 해야 하는가?**
>
> DOS 기반 레거시 시스템의 유지보수 인력이 점점 감소하고 있습니다. 시스템 장애 시 복구가 어려워지고 있으며, 매일 발생하는 수기 입력 오류로 인한 데이터 정합성 문제가 누적되고 있습니다. 지금 투자하지 않으면, 갑작스러운 시스템 장애 시 전체 계량 업무가 마비될 위험이 있습니다.

## 2. 현재 상황: 왜 변화가 필요한가?

> **스토리 의도**: 고객이 겪는 고통에 깊이 공감하고, 문제의 심각성 인식

### 2.1 현재의 도전과 고통

**"매일 반복되는 비효율, 그리고 쌓여가는 피로"**

2008년 도입된 현 계량대 시스템은 15년 이상 묵묵히 제 역할을 해왔습니다. 하지만 세상은 변했고, 이제 이 시스템은 오히려 업무의 발목을 잡고 있습니다.

| 문제 영역 | 현상 | 비즈니스 영향 |
|----------|------|--------------|
| **수기 입력** | 계량 후 전표를 손으로 작성하여 전달 | 오기입률 5%, 데이터 정합성 확인에 일 1시간 이상 소요 |
| **긴 대기 시간** | 운전기사가 자기 순번을 알 수 없음 | 평균 대기 45분, 기사들의 불만과 민원 지속 발생 |
| **유지보수 한계** | DOS 기반 시스템, 전문 인력 부족 | 장애 복구 지연, 시스템 개선 불가 |
| **정보 단절** | 실시간 현황 파악 불가 | 관리자가 현장에 직접 가야만 상황 파악 가능 |

### 2.2 변화하지 않으면?

> **현재 방식을 유지할 경우:**
> - DOS 전문 인력의 고령화로 장애 시 복구 불가 상황 발생 위험
> - 수기 전표 오류 누적으로 정산 분쟁 및 신뢰도 하락
> - 경쟁 업체 대비 운영 효율성 격차 확대
> - 운전기사 이탈로 인한 물류 파트너십 약화

### 2.3 변화했을 때의 미래

> **프로젝트 성공 후 모습:**
> - **계량팀**: "화면 하나로 3개 계량대 현황을 실시간으로 봅니다. 엑셀 수작업? 이제 버튼 한 번이면 됩니다."
> - **운전기사**: "앱에서 내 순번 보면서 차에서 편히 쉽니다. 내 차례 되면 알림이 와요."
> - **관리자**: "일간/월간 리포트가 자동으로 생성됩니다. 현황 파악하러 현장 갈 필요가 없어요."

## 4. 우리의 솔루션

> **스토리 의도**: "이렇게 해결합니다"를 명쾌하고 신뢰감 있게

### 4.1 솔루션 개요

**"손으로 쓰던 전표가 스마트폰으로, 눈으로 세던 순번이 자동 알림으로"**

```
기존 인디케이터(저울) 3대는 그대로 활용하면서,
데이터 수집과 처리를 완전히 현대화합니다.
운전기사는 앱으로, 관리자는 웹으로 - 모두가 편해집니다.
```

### 4.2 작업 범위

#### 포함 범위 (In-Scope)

| 구분 | 내용 | 고객 가치 |
|------|------|----------|
| **웹 대시보드** | 관리자용 실시간 모니터링, 통계, 차량/기사 관리 | 사무실에서 모든 현황을 한눈에 파악 |
| **모바일 앱** | 운전기사용 대기 순번, 푸시 알림, 전자 전표 | 차 안에서 편하게 대기, 종이 전표 불필요 |
| **Edge Server** | 기존 인디케이터 3대 연동, 데이터 자동 수집 | 수기 입력 제거, 오류 원천 차단 |
| **ERP 연동** | 일 마감 데이터 자동 전송 | ERP 수동 입력 업무 제거 |
| **전광판 연동** | 대기 순번 실시간 표시 | 앱 없는 기사도 순번 확인 가능 |

#### 제외 범위 (Out of Scope)

> 다음 항목은 본 프로젝트 범위에 포함되지 않습니다:
> - **인디케이터(저울) 하드웨어 교체** - 기존 장비 활용으로 비용 절감
> - **ERP 시스템 내부 수정** - 외부 연동 API만 활용
> - **사전 예약 기능** - Phase 2에서 검토 가능
> - **멀티 사이트 통합** - 향후 확장 시 고려
