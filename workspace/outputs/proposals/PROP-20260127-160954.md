## 1. 경영진 요약

### 핵심 메시지

부산공장 계량대는 현재 RFID 기반 수동 운영 방식의 한계에 직면해 있습니다. 종이 계량표 수취 시 안전 위험, 다중 배차 시 복수 RFID 카드 사용의 비효율, 특정 부서 전화 문의 집중으로 인한 업무 과중이 반복되고 있습니다.

**스마트 계량 시스템**은 LPR(차량번호 자동인식) + AI 기술, 모바일 APP, 전자 계량표를 결합하여 이 문제를 근본적으로 해결합니다.

| 핵심 지표 | 목표 |
|-----------|------|
| 무인 자동 계량 비율 | 90% 이상 |
| 종이 계량표 사용률 | 0% (Paperless) |
| 계량 처리 시간 단축 | 30% 이상 |
| 계량 에러율 감소 | 50% 이상 |
| 차량번호 인식률 | 95% 이상 |

### 투자 개요

| 항목 | 내용 |
|------|------|
| 프로젝트 기간 | 2026년 4월 ~ 11월 (8개월) |
| 총 투입 공수 | 498 MD (24.9 M/M), 버퍼 포함 598 MD (29.9 M/M) |
| 팀 구성 | 9명 (전담 6 + 부분 투입 3) |
| 방법론 | Hybrid (Waterfall 기반 + Agile 스프린트) |
| 포항공장 선행 경험 | 스마트 계량대(LPR) 운영 노하우 보유 |

## 2. 현재 상황과 도전 과제

### 2.1 현행 운영 방식

부산공장 계량대는 **RFID 스캔 → 종이 계량표 출력 → 수취** 방식으로 운영되고 있습니다. 이 방식은 계량 업무의 기본적인 흐름을 지원하고 있으나, 현장에서는 다음과 같은 구조적 문제가 반복적으로 발생하고 있습니다.

### 2.2 핵심 과제

#### 과제 1: 안전 위험 — 종이 계량표 수취 과정의 불안전 요소
차량기사가 계량대에서 종이 계량표를 수취하는 과정에서 안전하지 않은 상황이 발생합니다. 차량에서 내려 수취하거나, 움직이는 차량 근처에서 서류를 주고받는 동작은 현장 안전 관리의 사각지대입니다.

#### 과제 2: 운영 비효율 — 다중 배차 시 복수 RFID 카드 사용
하나의 차량에 여러 건의 배차가 발생할 경우, 각 배차 건별로 별도의 RFID 카드를 사용해야 합니다. RFID 카드 발급·관리·분실 대응에 불필요한 행정 비용이 지속적으로 발생합니다.

#### 과제 3: 업무 과중 — 특정 부서 전화 문의 집중
계량대에서 특이사항이 발생하면 특정 부서(물류관제실)에 전화 문의가 집중됩니다. 문의 유형과 관계없이 한 부서로 모든 통화가 몰리면서 해당 부서의 업무 과중이 심화되고 있습니다.

#### 과제 4: 정확도 한계 — 복잡한 계량 처리와 재계량 불가
1차/2차/3차 계량 처리가 복잡하여 에러 및 지연이 빈번하게 발생하고, 재계량이 불가능한 현재 구조에서는 오류 정정의 기회가 제한적입니다.

#### 과제 5: 시스템 노후화 — 분산된 배차/계량 관리 시스템
기존 배차 시스템과 계량 관리 시스템이 별도로 운영되어 데이터 일관성이 떨어지고, UI/UX가 노후화되어 사용자 불편이 가중되고 있습니다.

## 3. 변화하지 않으면?

현행 방식을 유지할 경우 예상되는 상황입니다.

| 영역 | 현재 상태 유지 시 예상 결과 |
|------|---------------------------|
| **안전** | 종이 계량표 수취 과정의 안전사고 위험 지속, 사고 발생 시 책임 이슈 |
| **비용** | RFID 카드 발급·관리·분실 대응 행정비용 누적, 종이 인쇄비 지속 발생 |
| **효율** | 수동 프로세스로 인한 계량 처리 지연, 피크 시간대 차량 대기 시간 증가 |
| **품질** | 계량 에러 지속, 재계량 불가로 인한 데이터 정확도 저하 |
| **인력** | 특정 부서 업무 과중 심화, 담당자 이직·번아웃 위험 |
| **확장성** | 물동량 증가 시 현행 시스템으로 대응 불가, 타 공장 확산 기반 부재 |
| **경쟁력** | 디지털 전환 지연으로 산업 내 경쟁력 저하 |

> 포항공장은 이미 스마트 계량대(LPR)를 운영하고 있습니다. 부산공장의 디지털 전환이 늦어질수록 공장 간 운영 격차가 벌어집니다.

## 5. 솔루션 개요

### 5.1 솔루션 구성

부산 스마트 계량 시스템은 **5개 핵심 모듈**로 구성됩니다.

```
┌─────────────────────────────────────────────────────────────┐
│                    부산 스마트 계량 시스템                       │

│ LPR+AI   │ 모바일   │  웹 관리  │ 계량대   │ 전자 계량표     │
│ 자동인식  │  APP     │  시스템   │ CS 프로그램│ & Paperless    │
├──────────┼──────────┼──────────┼──────────┼─────────────────┤
│ ▪ LPR    │ ▪ 배차   │ ▪ 배차   │ ▪ LPR   │ ▪ 전자 계량표   │
│   카메라  │   조회   │   등록   │   자동계량│   생성/조회     │
│ ▪ AI     │ ▪ 계량   │ ▪ 계량   │ ▪ 모바일 │ ▪ 카카오톡     │
```

### 5.3 모듈별 핵심 기능

#### 모듈 1: LPR + AI 차량번호 자동인식
- LPR 장치 1차 인식 + AI 시스템 2차 검증 (이중 확인)
- LiDAR 센서 차량 감지 → LPR 촬영 트리거 (자동)
- 차량번호 인식률 95% 이상 목표
- 인식 결과 → 배차 정보 자동 매칭 → 자동 계량

#### 모듈 2: 계량관리 모바일 APP (Flutter)
- 담당자/운전자 구분 로그인 (안전 로그인, 인증번호)
- 배차지시 현황 조회 및 다중 배차 선택 계량
- 모바일 OTP 보안 계량 (차량번호 오인식 시 대체 수단)
- 전자 계량표 조회 및 카카오톡/SMS 공유
- 배차/계량 완료 실적 상세 조회 (월별/일별)
- 계량실적 Push 알림 및 알림톡

#### 모듈 3: 계량 웹 관리 시스템 (React)
- 통합 배차 등록 관리 (부산물/폐기물/부재료/반출/일반)
- 실시간 계량 현황 대시보드 (WebSocket)
- 기준정보 관리 (운송사, 차량, 공통코드)
- 출문관리 기능
- 반응형 디자인, 고해상도/가변SIZE, 다양한 차트

#### 모듈 4: 계량대 CS 프로그램 (C# .NET)
- 인디게이터 연동 (RS-232C): 안정화 중량값 수신/저장
- LPR 자동계량, 모바일 OTP 계량, 수동 계량 3가지 모드 지원
- 초기화/재계량 기능 (기존 불가 → 신규 개발)
- 전광판, 자동차단기, 레이더센서, 차량검지기 등 H/W 제어
- 네트워크 장애 시 로컬 캐싱 후 자동 동기화

#### 모듈 5: 전자 계량표 및 유선통화 이원화
- 종이 계량표 100% 대체 → Paperless 운영
- 카카오톡, SMS를 통한 전자 계량표 공유
- 유선통화 이원화 (물류관제실, 자재창고 등 문의유형별 연결)

## 6. 기술 아키텍처

### 6.1 기술 스택

| 영역 | 기술 | 선정 사유 |
|------|------|----------|
| **백엔드** | Spring Boot 3.2 (Java 17) | PRD 제약조건 명시, 엔터프라이즈 안정성, 이중화 지원 |
| **프론트엔드 (Web)** | React 18 + TypeScript + Ant Design | 실시간 대시보드, 데이터 테이블(필터/정렬/소계/셀고정) |
| **모바일** | Flutter 3.x (Dart) | iOS/Android 크로스 플랫폼, 단일 코드베이스 |
| **계량대 CS** | C# .NET WinForms (VS 2022) | RS-232C 시리얼 통신 지원, 기존 계량 PC 호환 |
| **데이터베이스** | PostgreSQL 16 | ACID 트랜잭션, JSON 지원, 높은 확장성 |
| **캐시/세션** | Redis 7 | OTP 임시 저장(TTL 5분), 세션 캐싱, 이벤트 MQ |
| **웹 서버** | Nginx 1.24 | SSL 종단, 로드밸런싱, 정적 파일 서빙 |
| **인증** | JWT + Spring Security | Access Token(30분) + Refresh Token(7일), RBAC |
| **모니터링** | Prometheus + Grafana + ELK | 실시간 메트릭, 로그 중앙 관리 |
| **CI/CD** | Jenkins + Docker + Git | 자동 빌드/배포, Rolling Update |
