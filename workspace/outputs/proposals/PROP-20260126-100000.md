## 1. 경영진 요약

본 제안서는 귀사의 차량 계량대 시스템 현대화 프로젝트에 대한 기술 솔루션과 실행 계획을 제시합니다.

현재 15년 이상 운영된 DOS 기반 레거시 시스템을 웹 기반 관리 시스템과 모바일 앱으로 전환하여, 운전기사 대기 시간을 45분에서 20분 이하로 단축하고, 수기 전표를 100% 전자화하며, 데이터 오류율을 0.1% 이하로 개선합니다.

기존 인디케이터(저울) 장비를 그대로 활용하면서 엣지 서버를 통한 데이터 자동 수집, ERP 시스템 연동, 전광판 연동까지 포함하는 통합 솔루션을 제공합니다. React 기반 웹 대시보드와 Flutter 기반 크로스플랫폼 모바일 앱을 AWS 클라우드 인프라 위에 구축하여 99.5% 이상의 시스템 가용성을 보장합니다.

총 프로젝트 기간은 약 7개월(140일)이며, 14.5 M/M의 투입 공수가 예상됩니다.

## 2. 프로젝트 개요

### 2.1 배경

귀사의 현재 계량대 시스템은 2008년 도입 후 15년 이상 운영된 DOS 기반 레거시 시스템으로, 다음과 같은 문제점이 있습니다:

- 유지보수 인력 확보가 어려워 시스템 안정성 위협
- 수기 대장 작성 병행으로 인한 업무 비효율 및 데이터 오류 발생
- 운전기사 평균 대기 시간 45분으로 지속적인 민원 발생
- 실시간 현황 파악 및 통계 분석의 어려움

이에 기존 인디케이터 장비를 유지하면서 최신 기술 기반의 웹/모바일 시스템을 구축하여 업무 효율성과 고객 만족도를 획기적으로 개선하고자 합니다.

### 2.2 목표

- 웹 기반 관리 시스템 구축 (실시간 대시보드, 통계 리포트, 차량/기사 관리)
- 운전기사용 모바일 앱 개발 (대기순번 확인, 푸시 알림, 전자 전표)
- 기존 인디케이터 3대와의 RS-232 연동 및 데이터 자동 수집
- ERP 시스템 연동을 통한 일 마감 데이터 자동 전송
- 전광판 연동을 통한 대기 순번 실시간 표시
- 운전기사 대기 시간 단축 및 민원 해소

### 2.3 성공 기준

- 운전기사 평균 대기 시간 45분 → 20분 이하로 단축
- 수기 전표 발행 0건 (100% 전자화)
- 데이터 오류율 5% → 0.1% 이하
- 시스템 가용성 99.5% 이상
- 앱 화면 로딩 3초 이내

## 4. 솔루션 접근법

### 4.1 솔루션 개요

최신 기술 스택을 활용한 클라우드 기반 3-Tier 아키텍처와 엣지 컴퓨팅을 결합하여, 실시간 데이터 처리와 높은 가용성을 동시에 달성합니다. 기존 인디케이터 장비의 데이터는 현장 엣지 서버에서 수집하여 안정적으로 클라우드로 전송합니다.

### 4.2 시스템 아키텍처

시스템은 클라우드(AWS)와 현장(On-Premise) 두 영역으로 구성됩니다.

**클라우드 영역:**
- 웹 애플리케이션 (React SPA)
- API 서버 (FastAPI)
- 데이터베이스 (PostgreSQL - AWS RDS)
- 캐시 (Redis - AWS ElastiCache)
- 파일 저장소 (AWS S3)

**현장 영역:**
- 엣지 서버 (Ubuntu + Python)
- 인디케이터 3대 (RS-232 연동)
- 전광판 (TCP/IP 연동)

두 영역은 AWS Site-to-Site VPN으로 안전하게 연결됩니다.

### 4.3 기술 스택

- **Frontend (웹)**: React 18, TypeScript, TailwindCSS, Recharts
- **Frontend (모바일)**: Flutter 3, Dart, Riverpod, FCM
- **Backend**: FastAPI, Python 3.11, SQLAlchemy, Celery
- **Database**: PostgreSQL 15 (AWS RDS), Redis 7 (ElastiCache)
- **Infrastructure**: AWS (EC2, RDS, S3, CloudFront, VPN)
- **Edge**: Ubuntu 22.04, Python, pyserial, SQLite

### 4.4 개발 방법론

애자일(Agile) 방법론 기반 2주 스프린트 주기로 개발을 진행합니다.

- 2주 단위 스프린트로 점진적 기능 개발
- 매 스프린트 종료 시 데모 및 리뷰
- 지속적 통합/배포(CI/CD) 파이프라인 구축
- 고객 참여를 통한 요구사항 검증 및 피드백 반영
