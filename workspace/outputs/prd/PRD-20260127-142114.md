# 부산 스마트 계량 시스템 PRD (Product Requirements Document)

**버전**: 1.0
**작성일**: 2026-01-27
**상태**: Draft

---

## 1. 개요

### 1.1 배경

부산공장 계량대는 현재 RFID 스캔 후 종이 계량표를 출력하여 수취하는 방식으로 운영되고 있습니다. 이 과정에서 차량 운전자가 차량에서 내려 계량표를 수령해야 하는 불안전 요소가 존재하며, 다중 배차의 경우 동일 차량에 여러 개의 RFID 카드를 사용해야 하는 불편함이 있습니다.

또한 복잡한 계량(1차/2차/3차) 처리 과정에서 에러 및 지연이 발생하고, 재계량이 불가능한 구조로 인해 운영 효율성이 저하되고 있습니다. 계량대 특이사항 발생 시 특정 부서에 전화 문의가 집중되어 업무 과중 현상도 발생하고 있습니다.

이러한 문제를 해결하기 위해 LPR(License Plate Recognition) + AI 기반 차량번호 자동인식, 모바일 계량 APP, 전자 계량표를 도입한 스마트 계량 시스템을 구축하여 계량 프로세스의 자동화와 안전성을 확보하고자 합니다.

### 1.2 목표

- 차량번호 자동인식(LPR+AI) 기반 무인 계량 시스템 구축으로 계량 처리 자동화 달성
- Paperless 전자 계량표 도입으로 종이 계량표 출력/수취 프로세스 제거
- 모바일 APP 기반 계량 관리로 사용자 편의성 및 안전성 향상
- 기존 배차/계량관리 시스템 통합으로 운영 효율성 증대
- 계량 프로세스 단순화 및 재계량 기능 도입으로 정확도 향상

### 1.3 범위

#### 포함 범위
- LPR 차량번호 인식기 및 AI 검증 시스템 구축
- 스마트 계량 웹 시스템 개발 (배차등록, 기준정보, 계량관리, 출문관리)
- 계량관리 모바일 APP 개발 (iOS/Android)
- 계량대 프로그램(C/S) 개발
- 전자 계량표 시스템 개발
- 모바일 OTP 기반 보안 계량 기능
- 기존 시스템과의 통합 및 데이터 마이그레이션
- 사용자 교육 및 안정화 지원

#### 범위 외
- 타 공장(포항 등) 계량 시스템 구축
- ERP 시스템 전체 교체
- 계량대 물리적 구조 변경 (토목 공사)
- 기존 터치스크린 방식 완전 폐기 (병행 운영)

### 1.4 대상 사용자

| 사용자 유형 | 역할 및 주요 활동 |
|------------|------------------|
| 차량 운전자 | 계량대 진입, 모바일 APP으로 배차 선택 및 계량 진행, 전자 계량표 확인/공유 |
| 운송사 담당자 | 배차 정보 확인, 계량 실적 조회, 계량표 관리 |
| 계량 담당자 | 계량 모니터링, 수동 계량 처리, 이상 상황 대응 |
| 물류관제실 | 배차 등록, 계량 관리, 출문 관리, 전체 현황 모니터링 |
| 시스템 관리자 | 기준정보 관리, 사용자 권한 관리, 시스템 설정 |

### 1.5 성공 지표

- 계량 처리 시간 30% 단축 (기존 대비)
- 종이 계량표 사용량 100% 감소 (Paperless 달성)
- 차량번호 자동인식률 95% 이상 달성
- 계량 오류율 50% 감소
- 사용자 만족도 80% 이상 달성

## 2. 기능 요구사항 (Functional Requirements)

### FR-001: LPR 차량번호 자동인식
**우선순위**: HIGH
**설명**: LPR 장치를 통해 계량대에 진입하는 차량의 번호판을 자동으로 인식하고, AI 시스템을 통해 인식 결과를 검증하여 정확도를 높인다.

**User Story**: 차량 운전자로서, 나는 계량대에 진입할 때 차량번호가 자동으로 인식되기를 원한다, 수동으로 카드를 태깅하거나 번호를 입력하지 않아도 되기 때문에.

**Acceptance Criteria**:
- [ ] LPR 장치가 차량 진입 시 번호판을 자동 촬영한다
- [ ] AI 시스템이 LPR 인식 결과를 2차 검증한다
- [ ] 차량번호 인식률 95% 이상을 달성한다
- [ ] 인식 결과를 계량 서버로 실시간 전송한다
- [ ] 인식 실패 시 모바일 OTP 계량으로 자동 전환된다

### FR-002: 모바일 OTP 계량
**우선순위**: HIGH
**설명**: 차량번호 오인식 시 전광판에 표시된 OTP 번호를 모바일 APP에 입력하여 본인 인증 후 계량을 진행한다.

**User Story**: 차량 운전자로서, 나는 차량번호 인식이 실패해도 모바일 OTP로 계량을 진행할 수 있기를 원한다, 계량 지연 없이 업무를 수행하기 위해.

**Acceptance Criteria**:
- [ ] 차량번호 오인식 시 전광판에 6자리 OTP가 표시된다
- [ ] 운전자가 모바일 APP에서 OTP를 입력할 수 있다
- [ ] OTP 유효시간은 3분으로 제한된다
- [ ] OTP 인증 성공 시 등록된 전화번호 기반 차량 매칭이 이루어진다
- [ ] 인증 완료 후 정상 계량 프로세스가 진행된다

### FR-003: 계량관리 모바일 APP
**우선순위**: HIGH
**설명**: 운전자 및 담당자가 스마트폰을 통해 배차 조회, 계량 진행, 전자 계량표 확인 등의 업무를 수행할 수 있는 모바일 애플리케이션을 개발한다.

**User Story**: 차량 운전자로서, 나는 모바일 APP에서 배차 정보를 확인하고 계량을 진행할 수 있기를 원한다, 차량에서 내리지 않고 안전하게 업무를 처리하기 위해.

**Acceptance Criteria**:
- [ ] 담당자/운전자 구분 로그인이 가능하다
- [ ] 인증번호 기반 안전 로그인을 지원한다
- [ ] 배차 지시 현황 조회가 가능하다
- [ ] 다중 배차 선택 후 계량 진행이 가능하다
- [ ] 계량 완료 시 Push 알림을 받을 수 있다
- [ ] 전자 계량표 조회 및 공유(카카오톡, SMS)가 가능하다
- [ ] 배차/계량 실적 상세 조회(월별/일별, 기간설정)가 가능하다

### FR-004: 전자 계량표
**우선순위**: HIGH
**설명**: 종이 계량표를 대체하여 모바일 APP을 통해 전자 계량표를 확인하고 공유할 수 있는 기능을 제공한다.

**User Story**: 차량 운전자로서, 나는 계량표를 모바일로 받아 확인하고 필요 시 공유할 수 있기를 원한다, 종이 계량표를 받기 위해 차량에서 내리지 않아도 되기 때문에.

**Acceptance Criteria**:
- [ ] 계량 완료 시 전자 계량표가 자동 생성된다
- [ ] 모바일 APP에서 계량표 내용을 확인할 수 있다
- [ ] 카카오톡, SMS 등을 통해 계량표 공유가 가능하다
- [ ] 알림톡으로 계량 완료 안내가 발송된다
- [ ] 과거 계량표 이력 조회가 가능하다

### FR-005: 스마트 계량 웹 시스템
**우선순위**: HIGH
**설명**: 배차 등록, 기준정보 관리, 계량 관리, 출문 관리 등의 기능을 제공하는 웹 기반 관리 시스템을 개발한다.

**User Story**: 물류관제실 담당자로서, 나는 웹 시스템에서 배차 등록 및 계량 현황을 통합 관리할 수 있기를 원한다, 효율적인 업무 처리를 위해.

**Acceptance Criteria**:
- [ ] 부산물, 폐기물, 부재료, 반출, 일반 등 품목별 배차 등록이 가능하다
- [ ] 기준정보(차량, 운송사, 품목 등) 관리가 가능하다
- [ ] 실시간 계량 현황 모니터링이 가능하다
- [ ] 출문 관리 기능을 제공한다
- [ ] 다양한 차트와 대시보드를 제공한다
- [ ] 필터, 정렬, 소계, 셀고정 등 UI 편의 기능을 제공한다
- [ ] 다양한 디바이스 및 브라우저와 호환된다

### FR-006: 계량대 프로그램 (C/S)
**우선순위**: HIGH
**설명**: 계량대 현장에서 동작하는 클라이언트-서버 방식의 계량 처리 프로그램을 개발한다.

**User Story**: 계량 담당자로서, 나는 계량대에서 다양한 계량 방식(자동/모바일/수동)을 처리할 수 있기를 원한다, 상황에 맞는 유연한 계량 처리를 위해.

**Acceptance Criteria**:
- [ ] 안정화된 중량값을 자동 저장한다
- [ ] LPR 자동계량, 모바일 계량, 수동계량을 모두 지원한다
- [ ] 초기화 및 재계량 기능을 제공한다
- [ ] 인디게이터, LPR 및 관련 장비와 연동된다
- [ ] 전광판에 안내 메시지 및 OTP를 표시한다

### FR-007: 문의 유형별 통화 기능
**우선순위**: MEDIUM
**설명**: 계량대에서 문의 사항 발생 시 유형별로 담당 부서에 직접 연결되는 기능을 제공한다.

**User Story**: 차량 운전자로서, 나는 문제 발생 시 해당 담당 부서에 바로 연락할 수 있기를 원한다, 문의 전화가 특정 부서에 집중되는 것을 방지하기 위해.

**Acceptance Criteria**:
- [ ] 모바일 APP에서 문의 유형(물류관제실, 자재창고 등)을 선택할 수 있다
- [ ] 선택한 유형에 따라 해당 부서로 직접 통화 연결된다
- [ ] 긴급 문의와 일반 문의를 구분한다

### FR-008: 기존 터치스크린 병행 운영
**우선순위**: MEDIUM
**설명**: 스마트 계량 시스템 도입 후에도 기존 터치스크린 방식을 병행 운영하여 시스템 안정성을 확보한다.

**User Story**: 계량 담당자로서, 나는 스마트 계량 시스템 장애 시에도 기존 방식으로 계량을 처리할 수 있기를 원한다, 업무 연속성을 보장하기 위해.

**Acceptance Criteria**:
- [ ] 기존 터치스크린을 통한 수동 계량이 가능하다
- [ ] 스마트 계량과 수동 계량 간 전환이 원활하다
- [ ] 양쪽 시스템의 데이터가 통합 관리된다

## 3. 비기능 요구사항 (Non-Functional Requirements)

### NFR-001: 성능
- LPR 차량번호 인식 응답시간: 2초 이내
- AI 검증 처리시간: 1초 이내
- 웹 시스템 페이지 로딩시간: 3초 이내
- 모바일 APP 응답시간: 2초 이내
- 동시 접속 사용자: 100명 이상 지원

### NFR-002: 보안
- 모바일 APP 로그인 시 인증번호 기반 2차 인증
- OTP 기반 본인 인증 (유효시간 3분)
- 사용자별 권한 관리 (담당자/운전자 구분)
- 데이터 전송 시 SSL/TLS 암호화
- 개인정보(전화번호, 차량번호) 암호화 저장

### NFR-003: 확장성
- 향후 타 공장 확대 적용 가능한 아키텍처
- 계량대 추가 시 시스템 확장 용이
- 품목 및 프로세스 유형 추가 가능

### NFR-004: 가용성
- 시스템 가용률: 99.5% 이상
- 장애 발생 시 기존 터치스크린 방식으로 Failover
- 일일 데이터 백업

### NFR-005: 사용성
- 직관적인 UI/UX 제공
- 고해상도, 가변 SIZE 화면 지원
- 다양한 디바이스(PC, 태블릿, 모바일) 및 브라우저 호환
- iOS/Android 모바일 APP 지원

### NFR-006: 연동성
- 기존 인디게이터(계량기)와 RS-232C 연동
- LPR 장비와 실시간 연동
- 전광판, 차단기, 레이더센서 등 현장 장비 연동
- 카카오톡 알림톡 API 연동

## 4. 제약조건

### CON-001: 기술 제약
- **WEB**: Spring Boot 프레임워크 사용
- **WAS**: Spring Boot 내장 서버
- **DB**: PostgreSQL 사용
- **UI**: 웹은 React, 모바일은 Flutter 사용
- **C/S**: Visual Studio 2022 Professional로 개발
- **기존 장비**: 인디게이터, 계량시스템 PC, RS-232C 컨버터는 기존 장비 사용

### CON-002: 일정 제약
- 프로젝트 기간: 2026년 5월 ~ 10월 (총 6개월, 안정화 1개월 포함)
- 시스템 오픈 예정일: 2026년 11월
- 개발 점검회의: 월 1회 진행

### CON-003: 인프라 제약
- LPR 관련 장비 공사 필요
- 부산공장 계량대 1개소에 우선 적용
- 기존 계량대 물리적 구조 내에서 장비 설치

### CON-004: 외부 의존성
- LPR 차량번호 인식기 납품 및 설치 (외주)
- AI 차량번호 인식 솔루션
- 카카오 알림톡 API
- 모바일 앱스토어(App Store, Google Play) 배포
