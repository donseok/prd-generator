# 부산 스마트 계량 시스템 구축 PRD (Product Requirements Document)

**버전**: 1.0
**작성일**: 2026-01-27
**상태**: Draft
**출처**: 수행계획서(스마트 계량 시스템) - 개발팀Ⅰ, DMES팀 (2026.01)

---

## 1. 개요

### 1.1 배경
부산공장 계량대는 현재 RFID 스캔 후 계량표를 출력·수취하는 방식으로 운영되고 있다. 이 과정에서 종이 계량표 수취 시 안전하지 않은 요소(不 안전 요소)가 존재하며, 다중 배차 시 동일 차량에 여러 개의 RFID 카드를 사용해야 하는 비효율이 발생하고 있다. 또한 계량대 특이사항 발생 시 특정 부서에 전화 문의가 집중되어 업무 과중 현상이 나타나고 있다.

복잡한 1차/2차/3차 계량 처리로 인해 에러 및 지연이 빈번하게 발생하며, 재계량이 불가능한 구조로 인해 정확도 개선에 한계가 있다. 이러한 문제를 해결하기 위해 LPR(License Plate Recognition) + AI 기반 차량번호 자동인식, 모바일 계량 APP, 전자 계량표를 도입하는 스마트 계량 시스템 구축이 필요하다.

포항공장에서 이미 스마트 계량대(LPR) 운영 경험이 있으며, 이를 부산공장에 적용·확장하여 계량 프로세스의 자동화, 안전성 확보, 업무 효율 향상을 달성하고자 한다.

### 1.2 목표
- LPR + AI 기반 차량번호 자동인식을 통한 무인 계량 자동화 구현
- 모바일 APP 기반 전자 계량표 도입으로 Paperless 운영 달성
- 기존 배차/계량관리 시스템 통합 및 UI/UX 현대화
- 계량 프로세스 단순화로 에러율 감소 및 처리 속도 향상
- 재계량 기능 개발로 계량 정확도 향상

### 1.3 범위
#### 포함 범위
- LPR 차량번호 인식 시스템 (LPR + AI + 모바일 OTP 보안 계량)
- 스마트 계량 웹 관리 시스템 (배차등록, 기준정보, 계량관리, 출문관리)
- 계량 CS 프로그램 (계량대 프로그램 - 인디게이터/LPR 연동)
- 계량관리 모바일 APP (로그인, 배차조회, 계량진행, 전자계량표, OTP)
- 모바일 API (사용자/배차 정보 연동, 모바일 계량 처리)
- H/W 인프라 구축 (LPR, 전광판, 자동차단기, 레이더센서, 차량검지기, 인터폰)
- 유선통화 이원화 (물류관제실, 자재창고 등 문의유형별 통화 기능)
- 사용자 교육 및 안정화 모니터링

#### 범위 외
- 포항공장 계량 시스템 (이미 구축 완료)
- ERP 또는 MES 시스템 전면 교체
- 타 공장 확대 적용 (이번 프로젝트 범위 외)

### 1.4 대상 사용자
| 사용자 유형 | 역할 및 주요 활동 |
|------------|------------------|
| 차량기사 | 계량대 진입, 모바일 APP을 통한 배차 선택/계량 진행, 전자 계량표 확인 및 공유 |
| 운송사 | 배차지시 관리, 계량실적 조회, 모바일 Push 알림 수신 |
| 계량 담당자 | 배차등록, 계량관리, 기준정보 관리, 출문관리, 시스템 모니터링 |
| 물류관제실/자재창고 담당자 | 문의유형별 통화 대응, 계량 이상 처리 |
| 시스템 관리자 | 시스템 운영, H/W 모니터링, 사용자 관리 |

### 1.5 성공 지표
- 계량대 무인 자동 계량 비율: 90% 이상 (LPR+AI 자동인식 기반)
- 종이 계량표 사용률: 0% (전자 계량표 100% 전환)
- 계량 처리 시간 단축: 기존 대비 30% 이상 개선
- 계량 에러율 감소: 재계량 기능 도입으로 기존 대비 50% 이상 감소
- 차량번호 인식률: 95% 이상 (LPR+AI 기반, 환경적 요인 포함)
- 사용자 만족도: 80% 이상 (차량기사, 운송사, 계량담당자)

## 2. 기능 요구사항 (Functional Requirements)

### FR-001: LPR 차량번호 자동인식
**우선순위**: HIGH
**설명**: LPR 장치에서 차량번호를 1차 인식한 후 AI 시스템에서 차량번호를 재확인(검증)하여 자동 계량 실적을 처리한다. 라이다 센서(LiDAR) 인식 시 LPR 차량번호판을 촬영한다.

**User Story**: 차량기사로서, 나는 계량대에 진입하면 차량번호가 자동으로 인식되어 별도 조작 없이 계량이 진행되기를 원한다, 수동 RFID 스캔 없이 빠르고 안전하게 계량하기 위해.

**Acceptance Criteria**:
- [ ] LPR 장치가 진입 차량의 번호판을 자동 촬영한다
- [ ] AI 시스템이 LPR 촬영 결과를 검증하여 차량번호를 확정한다
- [ ] 라이다 센서(LiDAR) 감지 시 LPR 촬영이 트리거된다
- [ ] 차량번호 인식률 95% 이상을 달성한다
- [ ] 인식된 차량번호로 배차 정보를 자동 매칭한다
- [ ] 자동 계량 실적이 시스템에 저장된다

### FR-002: 모바일 OTP 보안 계량
**우선순위**: HIGH
**설명**: 차량번호가 오인식된 경우 전광판에 표시된 모바일 OTP 번호를 입력하여 모바일 계량을 진행한다. 등록된 전화번호로 차량번호를 인식하고 보안 OTP 검증을 수행한다.

**User Story**: 차량기사로서, 나는 차량번호가 오인식된 경우에도 모바일 OTP를 통해 안전하게 계량을 진행할 수 있기를 원한다, 계량 지연 없이 처리를 완료하기 위해.

**Acceptance Criteria**:
- [ ] 차량번호 오인식 시 전광판에 모바일 OTP 번호가 표시된다
- [ ] 모바일 APP에서 OTP 번호 입력으로 본인 인증이 가능하다
- [ ] 등록된 전화번호 기반 차량 매칭이 수행된다
- [ ] OTP 검증 완료 후 모바일 계량 실적이 처리된다
- [ ] 오계량 방지 보안 로직이 적용된다

### FR-003: 계량관리 모바일 APP
**우선순위**: HIGH
**설명**: 담당자/운전자 구분 로그인, 배차지시 현황 조회, 다중 배차 선택 계량, 전자 계량표 출력 및 공유, 배차/계량 실적 상세 조회 기능을 제공하는 모바일 APP을 개발한다.

**User Story**: 차량기사로서, 나는 모바일 APP에서 배차정보 확인, 계량 진행, 전자 계량표 수취를 한 번에 처리할 수 있기를 원한다, 종이 계량표 없이 효율적으로 업무를 처리하기 위해.

**Acceptance Criteria**:
- [ ] 담당자/운전자 구분 로그인이 가능하다 (안전 로그인, 인증번호)
- [ ] 배차지시 현황 조회 및 다중 배차 선택 계량이 가능하다
- [ ] 모바일 계량 진행이 가능하다
- [ ] 전자 계량표 출력 및 공유 기능 제공 (카카오톡, SMS 등)
- [ ] 전달사항 표시 기능이 제공된다
- [ ] 배차/계량 완료 실적 상세 조회 (월별/일별, 기간설정 가능)
- [ ] 계량실적 모바일 Push 알림 및 알림톡 확인

### FR-004: 계량 웹 관리 시스템
**우선순위**: HIGH
**설명**: 기존 배차/계량관리 시스템을 통합하여 웹 기반 관리 시스템을 구축한다. 부산물, 폐기물, 부재료, 반출, 일반 등 배차 등록 및 기준정보, 계량관리, 배차정보, 출문관리 기능을 제공한다.

**User Story**: 계량 담당자로서, 나는 통합된 웹 시스템에서 모든 배차 등록·관리 및 계량 업무를 처리할 수 있기를 원한다, 분산된 시스템 대신 하나의 시스템으로 효율적으로 관리하기 위해.

**Acceptance Criteria**:
- [ ] 부산물, 폐기물, 부재료, 반출, 일반 등 품목별 배차 등록이 가능하다
- [ ] 기준정보 관리 화면이 제공된다
- [ ] 계량관리 화면에서 실시간 계량 현황을 확인할 수 있다
- [ ] 배차정보 조회 및 관리가 가능하다
- [ ] 출문관리 기능이 제공된다
- [ ] UI/UX 솔루션 도입으로 고해상도, 가변 SIZE, 다양한 차트 화면 구성이 가능하다
- [ ] 다양한 디바이스 및 브라우저 호환이 지원된다
- [ ] 필터, 정렬, 소계, 셀고정 등 UI 편의 기능이 제공된다

### FR-005: 계량대 CS 프로그램
**우선순위**: HIGH
**설명**: 계량대에 설치되는 C/S 프로그램으로 안정화 중량값 저장, LPR 자동계량, 모바일 계량, 수동계량, 초기화/다시계량 처리, 인디게이터·LPR 및 관련 장비 수신 처리 기능을 수행한다.

**User Story**: 시스템 관리자로서, 나는 계량대 프로그램이 LPR, 인디게이터, 관련 장비와 안정적으로 연동되어 자동·수동·모바일 계량을 모두 지원하기를 원한다, 다양한 계량 상황에 대응하기 위해.

**Acceptance Criteria**:
- [ ] 인디게이터에서 안정화 중량값을 수신하여 저장한다
- [ ] LPR 자동계량 모드가 동작한다
- [ ] 모바일 OTP 기반 계량 모드가 동작한다
- [ ] 수동계량 모드가 동작한다 (기존 터치스크린 방식 병행 운영)
- [ ] 초기화 및 다시계량(재계량) 처리가 가능하다
- [ ] 인디게이터, LPR, 레이더센서, 차량검지기, 전광판, 자동차단기 등 H/W 수신 처리가 안정적으로 동작한다

### FR-006: 전자 계량표 및 Paperless
**우선순위**: MEDIUM
**설명**: 기존 종이 계량표를 대체하여 모바일 APP을 통한 전자 계량표 보기·공유 기능을 제공한다. 카카오톡, SMS 등을 통한 계량표 공유가 가능하다.

**User Story**: 차량기사로서, 나는 종이 계량표 대신 모바일에서 전자 계량표를 확인하고 필요 시 공유할 수 있기를 원한다, 종이 분실 위험 없이 안전하게 계량 증빙을 관리하기 위해.

**Acceptance Criteria**:
- [ ] 모바일 APP에서 전자 계량표를 조회할 수 있다
- [ ] 카카오톡, SMS 등을 통해 계량표 공유가 가능하다
- [ ] 계량 완료 후 자동으로 전자 계량표가 생성된다
- [ ] 계량표 이력이 기간별로 조회 가능하다

### FR-007: 유선통화 이원화 및 문의기능
**우선순위**: MEDIUM
**설명**: 기존 특정부서 전화 문의 집중 문제를 해결하기 위해 유선통화를 이원화(물류관제실, 자재창고 등)하고, 계량관리 APP에서 문의유형별 통화 기능을 추가한다.

**User Story**: 차량기사로서, 나는 계량대에서 문제 발생 시 적절한 담당 부서로 직접 연결되기를 원한다, 신속하게 문제를 해결하기 위해.

**Acceptance Criteria**:
- [ ] 계량관리 APP에서 문의유형별 통화 기능이 제공된다
- [ ] 유선통화가 물류관제실, 자재창고 등으로 이원화된다
- [ ] 문의유형에 따라 적절한 담당부서로 자동 연결된다

### FR-008: 모바일 API
**우선순위**: HIGH
**설명**: 사용자 및 배차 정보 송신, 모바일 계량 처리를 위한 API를 개발한다.

**User Story**: 개발팀으로서, 나는 모바일 APP과 웹 시스템 간 데이터 연동을 위한 안정적인 API가 필요하다, 실시간으로 정확한 배차·계량 정보를 제공하기 위해.

**Acceptance Criteria**:
- [ ] 사용자 정보 조회/인증 API가 동작한다
- [ ] 배차 정보 조회 API가 동작한다
- [ ] 모바일 계량 처리 API가 동작한다
- [ ] Push 알림 발송 API가 동작한다
- [ ] API 응답 시간이 2초 이내이다

## 3. 비기능 요구사항 (Non-Functional Requirements)

### NFR-001: 성능
- LPR 차량번호 인식 응답 시간: 3초 이내 (촬영 → AI 검증 → 결과 반환)
- 웹 시스템 화면 로딩: 3초 이내
- 모바일 APP API 응답 시간: 2초 이내
- 동시 계량 처리: 계량대 1기 기준 원활한 실시간 처리
- 계량 데이터 실시간 동기화: 지연 5초 이내

### NFR-002: 보안
- 모바일 OTP 기반 본인 인증 (오인식 시 보안 계량)
- 사용자별 권한 관리 (담당자/운전자 구분)
- 안전 로그인 (인증번호 기반)
- 통신 데이터 암호화 (LPR ↔ AI ↔ 계량서버)
- 개인정보 보호 (차량번호, 전화번호 등)

### NFR-003: 확장성
- 향후 추가 계량대 확장 용이한 아키텍처
- 품목 유형 추가 시 시스템 변경 최소화
- 다중 공장 확장 가능한 구조 (향후 고려)

### NFR-004: 가용성
- 시스템 가용률: 99% 이상 (계량대 운영시간 기준)
- 장애 복구 시간(RTO): 1시간 이내
- 데이터 복구 시점(RPO): 최소 1시간 이내
- H/W 장애 시 수동 계량(터치스크린) 모드 전환 가능
